## Prep Checklist for This Role

### Coding
FDE roles at Glean test coding but often emphasize problem-solving and communication over pure LeetCode difficulty. Expect medium-level problems.
- [ ] Arrays, hash tables, strings — all Must-Have
- [ ] Two pointers and sliding window
- [ ] Trees (BFS/DFS) — common in hierarchy / org-graph problems
- [ ] Data Structure Design — relevant to connector/index design problems

### System Design
Focus on enterprise search, AI grounding, and integration patterns.
- [ ] Enterprise search architecture (indexing, relevance, permissions)
- [ ] RAG pipeline (chunking, embedding, retrieval, re-ranking, generation)
- [ ] Connector / webhook integration patterns
- [ ] Multi-tenant data isolation and permissions enforcement
- [ ] OAuth 2.0 / SAML / SCIM — enterprise auth flows

### Behavioral / Customer Scenarios
FDE interviews often include situational questions — frame answers with customer impact.
- [ ] "Tell me about a time you worked directly with a customer to solve a technical problem"
- [ ] "Describe a time you had to translate vague requirements into a concrete solution"
- [ ] "How do you balance customer requests vs. product roadmap priorities?"
- [ ] "Tell me about a complex integration you built"

### Product Sense
- [ ] Know Glean's core product: universal search, AI answers, knowledge graph
- [ ] Know which connectors/integrations matter most to enterprise (Slack, Drive, Jira, Confluence, GitHub, Salesforce)
- [ ] Be ready to walk through: "How would you onboard a new enterprise customer onto Glean?"

### Project Deep Dive
- [ ] Prioritize any work involving: customer-facing systems, integrations/APIs, search, or LLM-powered products
